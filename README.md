# 阿瓦隆人格测试（Avalon）

静态前端小项目：答题 → 按**八维累计分**（每题四选一，选项对不同维度加 **1/2/3…** 分）为每个**角色原型**算加权分 → 取最高分为主结果，并生成风格轴与分享卡片。

---

## 一、当前状态与需注意点（自行维护时）

| 项目 | 说明 |
|------|------|
| 技术形态 | 纯 HTML/CSS/ES 模块，无后端；计分在浏览器内完成。 |
| 题库位置 | `js/questions.js` 中 `QUESTIONS` 数组。 |
| 题量硬编码 | `index.html` 里首页副标题、`meta`、首帧进度条等处写死了 **「15」道题**（与当前题库一致）；若增删题目，请同步改掉这些文案与 `aria-valuemax` 等，避免误导。逻辑进度仍由 `QUESTIONS.length` 驱动。 |
| 接近度百分比 | `roleMatchPercent` = 你的加权分 ÷「在该题库下该角色理论上能拿到的最高加权分」×100，因此**改题后**各角色的「气质接近度」刻度会随新题变化，必要时可用 `scoring.js` 自测或微调角色 `weights`。 |
| 画布与图片 | 分享海报依赖 `canvas` 与 `pictures/` 内资源；用 `file://` 打开时可能无法导出图片（浏览器安全策略），建议通过 HTTP 访问。 |
| 结果立意 | 文案为**娱乐向社交博弈风格**，不对应真实桌游阵营或能力。 |

## 二、得分逻辑（核心）

### 2.1 维度累计分（每题一个选项）

- 在 `js/roles.js` 中定义 **8 个维度**（英文名，题库 `scores` 的键名必须与其完全一致）：

  `Logic`, `Intuition`, `Active`, `Reserved`, `Lead`, `Support`, `Direct`, `Hidden`

- 每题用户选一个选项；该选项的 **`scores` 为对象**，形如 `{ Logic: 2, Reserved: 1 }`，表示对 `Logic` **+2**、`Reserved` **+1**，其余维度不当场增加。

- 分值一般为 **1（轻）/2（中）/3（重）**，也可用其他正数；「上一步」会按同一对象**原样扣回**。

- 全程维护各维度总分对象 `counts`（见 `emptyCounts` / `addScores` / `subtractScores`）。

### 2.2 角色匹配（主结果 / 次结果）

- 每个角色在 `ROLES` 里有一份 **`weights`**（非负整数），表示该角色更「吃」哪些维度。

- **角色加权分**：

  \[
  \text{score}(\text{role}) = \sum_{d \in \text{维度}} \text{weights}[d] \times \text{counts}[d]
  \]

- 对所有角色算分后 **降序**排序；**第一名**为 `primary`，**第二名**为 `secondary`（用作次要接近度展示）。

- **平分**：比较同一加权分时，按 `TIE_BREAK_ORDER` 中角色名**靠前者优先**（见 `js/roles.js` 顶部）。

### 2.3 风格维度（结果页四条滑轴）

- 与「八维计数」不同，结果页的「推理/直觉」等四条轴是 **把两两维度计数合成一条 0–100 的连续谱**（`profileAxesFromCounts`）：

  | 轴 | 左端计数 | 右端计数 | 含义（偏右 = 更靠右标签） |
  |----|-----------|-----------|---------------------------|
  | 推理 ↔ 直觉 | `Logic` | `Intuition` | 右端占比高则更偏直觉 |
  | 静观 ↔ 积极 | `Reserved` | `Active` | |
  | 辅助 ↔ 带队 | `Support` | `Lead` | |
  | 直白 ↔ 隐秘 | `Direct` | `Hidden` | |

- 若某轴左右计数**均为 0**，该轴显示为 **50（居中）**。

- 展示文案规则（与 `main.js` 一致）：**正中 50** 显示「居中」；偏一侧时用「主导强度 % + 偏某标签」（避免在极端端出现难看的 0% 叙述）。

### 2.4 「气质接近度」百分比

- 对**角色 r**：令 `actual = score(r)`，`cap =` 在**当前 `QUESTIONS` 下**，**每题**在四个选项中取「对 r 的加权贡献」最大者，再**跨题求和**（见 `maxTheoreticalWeightedScoreForRole`）。加权贡献 = \(\sum_d \text{weights}[d] \times \text{选项在该维度的加分}\)。

- 展示度：`min(100, round(100 * actual / cap))`。

## 三、结果类型与「对标」关系（八角色）

角色是**气质原型**，与桌游身份同名仅为风格隐喻；匹配完全由 **weights + 你的维度计数** 决定。

| `id` | 展示名 | 权重侧重（摘要，详见 `roles.js`） |
|------|--------|----------------------------------|
| `merlin` | 梅林 | 推理、静观、隐秘偏高；带队/辅助/直白占一定比重 |
| `percival` | 派西维尔 | 直觉、辅助、直白；兼有一点静观/隐秘 |
| `loyal` | 忠臣 | 辅助、直白、推理；稳态与配合 |
| `arthur` | 亚瑟 | 带队、积极、直白 |
| `morgana` | 莫甘娜 | 隐秘、积极、直觉 |
| `assassin` | 刺客 | 积极、直白、带队 |
| `mordred` | 莫德雷德 | 隐秘、静观、推理 |
| `oberon` | 奥伯伦 | 直觉与隐秘双高，其余维度较匀 |

若要「某类玩家更容易测出某角色」，应 **调该角色的 `weights`**，或 **改题目的 `scores` 分布**，使二者一致。

## 四、每条结果里有哪些「描述字段」

每条 `ROLES` 一项通常包含（用于结果页 / 分享 / 图）：

| 字段 | 用途 |
|------|------|
| `name` | 中文名；排序平局、接近度、分享标题等 |
| `portraitFile` | `pictures/` 下文件名 |
| `weights` | 计分用 |
| `title` | 结果页主标题 |
| `subtitle` | 副标题 / 一句话 |
| `keywords` | 标签 chips |
| `body` | 「结果解读」长文（可含 `\n`） |
| `pros` / `cons` | 「你的优势」「你的盲区」列表 |
| `shareShort` | 不含名衔的短分享句段；复制全文时在 `main.js` 里还会拼上品牌与网址 |

分享海报上的短文案会用到 `shareShort`（`poster.js` 内会去掉换行再画）。

## 五、如何自行补充或修改题目

1. 编辑 **`js/questions.js`**：每题 **四个选项**（`A`–`D`），结构示例：

   ```js
   {
     id: 16,
     question: "题干字符串",
     options: [
       { key: "A", text: "轻度倾向推理+静观", scores: { Logic: 1, Reserved: 1 } },
       { key: "B", text: "更冲一点", scores: { Active: 2, Lead: 1 } },
       { key: "C", text: "很重直觉", scores: { Intuition: 3 } },
       { key: "D", text: "无厘头但也算分", scores: { Intuition: 1, Support: 1, Hidden: 1 } },
     ],
   },
   ```

2. `scores` 的**键**须为 `DIMENSIONS` 成员；**值**为本次加分数（建议正整数）。

3. **若总题数变化**：同步修改 **`index.html`** 里写死的题数与 `aria-valuemax`、首页/`meta` 文案。

4. 改完后建议实机跑一局，必要时微调 `roles.js` 的 `weights` 或各题分值，避免某角色过于垄断或接近度全挤在中间。

## 六、关键文件速查

| 文件 | 作用 |
|------|------|
| `js/questions.js` | 题库 |
| `js/roles.js` | 维度列表、平局序、角色权重与全部结果文案 |
